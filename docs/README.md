## 维护原则

- **不要重复**：同一信息只在一个文件里写。比如对阵表算法只在 6-核心算法.md 里写，其他地方引用即可。
- **不要拆太碎**：合并比拆分优先。新增内容能塞进现有文件就塞进去，不要随便建新文件。
- **改动同步归档**：大版本变更（v1.2 → v1.3 等）时，把当前所有 `1-` 到 `6-` 文件打包快照到 `归档/`。
- **决策有"复议触发"**：写决策不要只写结论，要写"什么情况下重新考虑"。
- **中文文件名 + 数字前缀**：保证排序自然 + 直观易读。
