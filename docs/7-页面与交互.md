## 设计原则

| 原则 | 说明 |
|---|---|
| **工具优先** | 每个 Tab 首屏都要让"最常做的事"占黄金位（首屏第一屏内可达） |
| **智能形态切换** | 同一个 Tab 根据用户状态切换 2-4 种布局，不是死板的固定页 |
| **不堆 banner 轮播** | 工具型小程序不需要花哨展示，直接给任务卡片 |
| **召集人零门槛** | "创建活动"按钮始终一键可达，不超过 1 次点击 |
| **新用户引导明确** | 只给 2 个清晰选择，不做多步 onboarding |
| **空状态有出路** | 任何"暂无数据"页都要给下一步动作建议 |
| **分享卡片可独立读懂** | 群里点开卡片，不打开小程序也能看懂活动核心信息 |

## 默认 Tab 与状态判断

### 默认 Tab
- 用户主动打开小程序 → 默认进入 **活动 Tab**
- 用户从微信群分享卡片进入 → **直接跳活动详情/对阵表/段位报告**，不经过 Tab

### 活动 Tab 首屏形态判断逻辑

```
if (新用户 && 0 活动):
    → 形态 2（新用户引导）
elif (当天已有活动报名/组织):
    → 形态 1（今天的活动）
elif (无自己组织过的历史活动):
    → 形态 3（散兵球友：附近活动 + 创建引导）
else:
    → 形态 4（老用户无活动：发现 + 训练打卡）
```

## 全局组件规范

### 9.1 顶部 bar
- 高度：44pt（标准小程序）
- 含返回按钮 + 页面标题 + 右上操作按钮（如分享、设置）

### 9.2 底部 Tab bar
- 5 个 Tab：🏸 活动 / 🏆 比赛 / 📊 段位 / 💪 训练 / 📚 教学
- 高度：50pt + 安全区
- 选中状态：颜色高亮 + 图标 filled

### 9.3 按钮规格

| 类型 | 用途 | 样式 |
|---|---|---|
| 主按钮（Primary） | 核心动作（创建活动/报名/打卡） | 大、高亮、占满宽度 |
| 次按钮（Secondary） | 辅助动作（发现附近/分享） | 中、描边 |
| 文字按钮（Text） | 跳转/查看更多 | 小、无边框 |
| 危险按钮（Danger） | 删除/移除 | 红色、需二次确认 |

### 9.4 卡片规格
- 圆角：8px
- 阴影：轻微，不夸张
- 内边距：16px
- 卡片间距：12px

### 9.5 海报规格（分享用）
- 尺寸：1080×1920（朋友圈竖图）或 750×750（群分享方图）
- 必含：YMQ Logo、二维码、核心信息
- 生成方式：后端生成 PNG（保证微信扫码可识别）

### 9.6 加载状态
- 列表 loading：骨架屏（不要转圈圈）
- 按钮 loading：按钮内 spinner + 文字"提交中..."
- 全页 loading：仅首次进入时显示

### 9.7 空状态
任何空列表都要给：
- 一句话说明"为什么是空的"
- 一个清晰的下一步动作按钮

例：
> "你还没创建过活动"
> [➕ 创建第一场活动]

### 9.8 错误状态
- 网络错误：可重试按钮
- 权限错误：跳转登录或授权
- 业务错误：明确说明错误原因 + 建议动作
