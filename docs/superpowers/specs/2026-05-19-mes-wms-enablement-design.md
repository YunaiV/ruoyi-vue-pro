# MES & WMS Module Enablement Design

**Goal:** Fix MES 404 ("请求地址不存在") and WMS 500 ("服务器错误,请联系管理员!") errors, enabling all functional menus in both modules.

**Architecture:** The project uses a multi-module Maven build (`ruoyi-vue-pro`) with optional business modules. MES is commented out from the build; WMS is active but missing database tables. Fix requires: (1) create missing DB tables with MySQL-compatible DDL, (2) re-enable MES in Maven build, (3) rebuild JAR, (4) restart backend.

**Tech Stack:** Maven, Spring Boot 3.2, MyBatis-Plus, MySQL 8.0, Vue 3 + Element Plus

---

## Background

### MES Module (yudao-module-mes)
- **Backend code:** Fully implemented (~93 controllers, ~95 services, ~90 DOs, ~90 mappers) in `yudao-module-mes/`
- **Frontend code:** Fully implemented (~250 .vue files, ~85 API .ts files) in `yudao-ui-admin-vue3/`
- **Menu/dict data:** Already seeded in `sql/mysql/ruoyi-vue-pro.sql`
- **Problem:** Module excluded from Maven build (commented out in root `pom.xml` and `yudao-server/pom.xml`), and `~32 database tables do not exist`
- **Error:** `admin-api/mes/home-statistics/work-order-status` returns 404 because the backend controller class is not compiled into the JAR

### WMS Module (yudao-module-wms)
- **Backend code:** Fully implemented (18 controllers, 18 services, 16 DOs, 17 mappers) in `yudao-module-wms/`
- **Frontend code:** Fully implemented (40 .vue files, 17 API .ts files) in `yudao-ui-admin-vue3/`
- **Menu/dict data:** Already seeded in `sql/mysql/ruoyi-vue-pro.sql`
- **Problem:** Module is active in the Maven build, but `14 database tables do not exist`
- **Error:** Any WMS page call hits a controller, which queries a non-existent table via MyBatis-Plus, producing a 500 error

### Common Issue
Both modules have their DDL defined in `src/test/resources/sql/create_tables.sql`, written in H2 syntax (`GENERATED BY DEFAULT AS IDENTITY`, double-quoted identifiers). These must be converted to MySQL syntax before execution.

---

## Implementation Plan

### Phase 1: Database Tables
- Convert H2 DDL to MySQL DDL for both modules
- Execute the converted SQL against the `ruoyi-vue-pro` MySQL database
- Verify all tables created successfully

### Phase 2: Maven Build Configuration
- Uncomment `<module>yudao-module-mes</module>` in root `pom.xml`
- Uncomment `yudao-module-mes` dependency in `yudao-server/pom.xml`

### Phase 3: Rebuild & Restart
- Run `mvn clean package -Dmaven.test.skip=true` to rebuild backend JAR
- Restart backend service on port 48080
- Verify MES home API (`/admin-api/mes/home-statistics/work-order-status`) returns 200
- Verify WMS home API (`/admin-api/wms/home-statistics/order-summary`) returns 200

### Phase 4: Frontend Verification
- Test MES menu navigation from the browser
- Test WMS menu navigation from the browser
- Verify CRUD operations on key sub-menus (warehouse, item, orders)
