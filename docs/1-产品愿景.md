## 产品定位

### 一句话
**「让一群人一起打球这件事，从拉群到散场，全程顺滑。」**

### 详细描述
服务业余羽毛球社群的微信小程序，5 大核心功能解决业余球友全场景痛点：

- **活动**：召集人不用再手工排对阵表；缺人时能让附近球友看到并申请加入
- **比赛**：内部比赛能像专业赛事一样有比分页、有战报
- **段位**：5 分钟测出 13 级段位；AI 教练点评握拍站位
- **训练**：测完段位有个性化提升计划，4 周后复测看进步
- **教学**：精选教程库支撑训练计划，并对外做 SEO

### 差异化

| 现有产品 | 主打 | 痛点 |
|---|---|---|
| 约球 App（约球、热门） | 陌生人约球 | 熟人圈不用、监管严、女性流失 |
| 全民羽球 | 个人数据 | 启动重、APP 门槛 |
| 微信群 + 群接龙 | 组织活动 | 只能接龙，不能排对阵 |
| **YMQ（我们）** | **熟人球队 + 高频组织工具 + 活动公开化** | **填补"群"和"App"之间的空白** |

**核心定位**：不和"陌生人约球 App"竞争，做"微信群里的对阵表生成器 + 战报机器"，缺人时再把活动公开给附近球友。用户来自微信群，回到微信群——我们是中间的工具层。

### 小程序底部 Tab 结构

```
[ 🏸 活动 ]  [ 🏆 比赛 ]  [ 📊 段位 ]  [ 💪 训练 ]  [ 📚 教学 ]
```

## 北极星指标

**WAO（Weekly Active Organizer）**：每周至少创建 1 场活动的用户数。

理由：召集人是产品杠杆，1 个召集人能带来 6-12 个参与者。

### 阶段目标

| 阶段 | 时间 | 关键指标 |
|---|---|---|
| MVP 验证 | 上线后 1 个月 | WAO ≥ 20，总用户 ≥ 300 |
| 种子增长 | 上线后 3 个月 | WAO ≥ 200，总用户 ≥ 5000，周留存 ≥ 35% |
| 冲月活破万 | 上线后 6 个月 | MAU ≥ 10000，WAO ≥ 800，K 系数 ≥ 1.2 |

### 辅助指标

- 接龙→对阵表生成转化率 ≥ 80%
- 分享卡片打开率 ≥ 25%
- 段位评测完成率 ≥ 70%
- 公开活动占比 ≥ 30%
- 附近活动申请通过率 ≥ 50%
- 训练计划生成→打卡转化率 ≥ 40%
- 训练计划 4 周完成率 ≥ 20%
- 教学 Tab 周打开率 ≥ 30%

## 用户画像

### A：球队队长 / 活动召集人（KOC，MAU 占比 10%，贡献 90% 活动）
- **特征**：30-45 岁，男性偏多，单位/小区/培训班队长
- **痛点**：接龙后人数不对要重排、新人加入要解释规则、要照顾男女搭配水平均衡、临时缺人只能在多个群发广告
- **诉求**：**省事**

### B：普通球友（MAU 占比 75%）
- **特征**：20-50 岁，男女均衡，每周打 1-2 次
- **痛点**：不知道自己什么时候上场、想知道今天和谁组队、想看自己打了多少场
- **诉求**：**少操作**

### C：进阶玩家 / 比赛参与者（MAU 占比 15%）
- **特征**：业余 5 级以上，热衷比赛
- **痛点**：比赛没有专业记分系统、想看水平历史变化、想知道段位
- **诉求**：装备党 / 数据控

### D：散兵球友（MAU 占比 5%，增长潜力大）
- **特征**：刚搬到新城市/没固定球队/朋友圈缺球友
- **痛点**：想找附近活动加入但不知道在哪、加陌生人微信尴尬、怕被骗
- **诉求**：**安全、可信地融入附近球友圈**（通过"活动"功能的"发现附近"子能力）

### E：想自学提升的球友（v1.2 新增，MAU 占比预估 20-30%）
- **特征**：跨段位（D-B 都有），有提升欲望但缺方法
- **痛点**：B 站抖音视频太碎、不知道自己该练什么、练了不知道有没有进步
- **诉求**：**有人告诉我练什么 + 看得到进步**（通过"段位+训练+教学"三件套）

## 核心用户旅程

### 旅程 1：召集人组织一场活动（最高频）

```
群里发"我有场地，今晚 7-9 点"
→ 召集人打开 YMQ 创建活动
→ 生成接龙卡片，分享回群
→ 群友点链接 → 微信授权 → 自动加入
→ 召集人到点点"开始排对阵"
→ 系统按性别/水平/上场次数均衡 → 生成 X 轮对阵表
→ 召集人确认 → 一键转发"对阵表卡片"回群
→ 每人看自己"7:00 第 1 场 / 7:30 第 2 场 / 8:00 休息"
→ 打完点"散场" → 自动生成战报海报
```

### 旅程 2：散兵球友通过"活动 Tab→发现附近"加入活动

```
新球友打开小程序 → 底部 Tab "活动" → 切到"发现附近"
→ 授权位置 → 看到附近 3km 内"今晚还缺人"的公开活动
→ 点活动卡片 → 申请加入（填段位 + 申请理由）
→ 召集人审核（看资料 + 信用分）→ 通过
→ 球友收到"申请通过" → 自动出现在对阵表
→ 当晚现场打球 → 第二天信用分 +1
```

### 旅程 3：段位评测裂变破圈

```
群里有人发"我测出来是 4.5 级" → 朋友点链接
→ 10 道选择题 + 5 个动作选项 → 5 分钟完成
→ 出报告卡片："你是 4.3 级·进攻型"
→ "查看详细分析"需分享朋友圈或群解锁
→ 解锁后看到 9 张精美图：水平定位/技术雷达图/进阶路线
```

### 旅程 4：办一场内部比赛

```
召集人 → "比赛"Tab → 创建 → 选「接力赛 / 淘汰赛」
→ 接力赛：分两队、按顺序派人、累计 100 分
→ 淘汰赛：选 8/16/32 人、自动种子签 + 对阵图
→ 裁判端：手机扫码做裁判，点比分
→ 观众端：群里点链接看实时比分
→ 决出冠军 → 生成颁奖海报
```

### 旅程 5：测完段位 → 看教学 → 跟训练计划 → 复测看进步

```
段位评测完成 → 系统推送"要看看 4 周提升计划吗？"
→ 进入"训练"Tab → 看到 4 周菜单（针对弱项）
→ 每天打开打卡 → 看"教学"Tab 推荐视频 + 做训练菜单
→ 4 周后系统提示"复测看进步"
→ 复测对比海报：4 周前 C2·65 → 现在 C3·78
→ 强烈鼓励分享朋友圈
```

## 功能一：活动

> 产品最核心、最高频的功能。代码层面由 `ymq-module-activity`（创建/接龙/对阵）+ `ymq-module-social`（信用/黑名单/举报/实名）协作实现。

### 6.1 角色
- **召集人**：发起活动、确认对阵、结束活动
- **参与者**：接龙加入、看自己的对阵
- **散兵球友**：通过"发现附近"申请加入公开活动
- **观众**：未登录用户可看活动公开页（吸引注册）

### 6.2 活动创建表单

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 活动名称 | 文本 | 是 | 默认"YYYY-MM-DD 晚 7 点·XX 场馆" |
| 场馆名称 | 文本 | 是 | 可保存常用场馆 |
| 场馆地址 | 文本+地图 | 否 | 用于导航 + 附近发现 |
| 开始/结束时间 | 时间 | 是 | |
| 场地数 | 数字 | 是 | 1-10 片 |
| 计划人数 | 数字 | 是 | 默认 6 人 |
| 活动模式 | 单选 | 是 | 男双/女双/混双/自由组合 |
| 轮换规则 | 单选 | 是 | 固定搭档/每轮换搭档 |
| 单场用时 | 数字 | 是 | 默认 15 分钟 |
| AA 费用 | 数字 | 否 | 用于二维码收款 |
| **对外公开** | **开关** | **否** | **开启后被附近球友发现，详见 §6.7** |

### 6.3 接龙报名

- 参与者点"我要报名"→ 微信授权 → 一秒加入
- 参与者可备注"我大概晚 30 分钟到"
- 召集人可设候补名单
- 召集人可移除某人（带原因，对方收通知）
- 距开始 2h 未满员，召集人收提醒"是否取消/继续/对外公开"

### 6.4 对阵表生成（核心算法见 [6-核心算法.md §1](6-核心算法.md)）

**输入**：参与者列表（性别/水平）、场地数、单场时长、总时长、模式

**约束（按优先级）**：
1. 每场 4 人（双打）
2. 同一轮每人最多上一场
3. 混双：每队严格 1 男 1 女
4. 每人上场次数差 ≤ 1
5. 避免连续上场超过 2 轮
6. 同两人不连续两轮组队

**召集人可手动**：拖动换位、锁定某场对阵、一键重新生成（保留锁定项）

### 6.5 对阵表分享卡片
- 活动名 / 时间 / 场馆
- 每轮对阵简表（场地 1 张三+王五 vs 李四+赵六 / 休息：周七）
- 二维码（扫码看自己的对阵）

### 6.6 散场后流程
- 召集人点"结束活动" → 算每人上场次数、参与时长
- 跳记分（功能二）或直接生成战报
- 战报海报：今日 MVP、最佳搭档、黑马、上场最多

### 6.7 发现附近活动（子功能）

> **核心定位**：不是"找人"，是"找球打"——让缺人的活动能被附近球友看到。
> **入口位置**：活动 Tab 下方"发现附近"子页。

#### 与"约球 App"的关键差异

| 维度 | 约球 App | 我们 |
|---|---|---|
| 主角 | 人 | 活动 |
| 入口 | 看附近的人 → 加好友 | 看附近缺人活动 → 申请加入 |
| 关系 | 私聊慢慢熟 | 加入活动现场认识 |
| 信任 | 个人头像 | 召集人审核 + 信用分 |
| 监管类目 | 社交·交友（严） | 体育·赛事（宽） |

#### 公开活动开关详解

| 字段 | 默认 | 说明 |
|---|---|---|
| 对外公开 | 关 | 开启后可被陌生球友发现 |
| 可见半径 | 3km | 1km / 3km / 5km / 10km |
| 接受陌生人申请 | 开 | 关闭则只可见不可申请 |
| 申请需审核 | 开 | 关闭则直接加入（不推荐） |
| 限制段位 | 不限 | 例：仅 C1-B2 |
| 限制性别 | 不限 | 缺男/缺女 |
| 通过陌生申请上限 | 1 | 防止陌生人占多数 |

#### 发现页设计
- 顶部：定位区 + 筛选
- 中部：地图/列表切换
- 卡片：活动名 + 距离 + 缺人数 + 召集人段位 + 距开始 + 申请按钮
- 排序权重 = 距离 + 时间临近度 + 缺人紧迫度 + 召集人信誉（算法见 [6-核心算法.md §8](6-核心算法.md)）

#### 申请流程

**球友视角**：
```
发现页点卡片 → 看详情 → 申请加入
→ 填段位 + 申请理由 → 提交进入"待审核"
→ 通过：订阅消息通知 + 自动加入对阵表
→ 拒绝：通知附原因
```

**召集人视角**：
```
活动详情顶部"待审核 (X)"红点
→ 看申请人段位 + 信用分 + 历史活动数 + 距离
→ 1 键通过 / 拒绝
```

#### 信任与安全机制

| 机制 | 说明 |
|---|---|
| 实名要求 | 申请加入必须微信实名（绑 phone） |
| 信用分 | 默认 100，放鸽子 -20，准时 +1，被投诉 -30（详见 [6-核心算法.md §9](6-核心算法.md)） |
| 黑名单 | 召集人可加 5 人；信用 < 60 进系统灰名单 |
| 首次惩罚轻 | 新用户首次放鸽子只警告不扣分 |
| 举报机制 | 任何用户可举报，运营 24h 内处理 |
| 女性保护 | 女性默认开启"仅活动内可见" |

#### 召集人激励
- 公开活动多展示"⭐ 公开"标签
- 召集人成长等级（已办活动数、好评率），LV5 解锁高级头衔
- 公开活动满员快 → 推送"附近还有 5 人在看你的活动"

#### 数据隐私
- 用户位置只存"最近进入发现页时的脱敏坐标"（精度到街道）
- 不持续追踪
- 用户可清除位置记录

## 功能二：比赛

### 7.1 接力赛 100 分

**规则**：两队对战，按顺序进行 N 项（男单/女单/男双/女双/混双），每局 21 分，分差累计到 100 分先到者胜。

**裁判端**：大按钮加分（误触少），长按减分。自动判断每局胜负、自动进入下一项、累计到 100 分自动锁分。

**观众端**：同样比分大屏，无操作。WebSocket 实时同步。可分享"围观比分"链接。

### 7.2 淘汰赛（4/8/16/32 人单淘汰）

**种子签**：
- **SNAKE 蛇形**（默认）：按水平降序分散到不同 1/4 区
- **RANDOM 随机**
- **MANUAL 手动**

**对阵图**：经典淘汰赛树形图，横向滚动。每场点击进入裁判端记分。

**非 2^N 人数**：上升到最近的 2 的幂，多余位置给种子排名前的选手轮空。

### 7.3 颁奖海报
冠亚军确认后自动生成精美颁奖图。

## 功能三：段位评测 + AI 教练

### 8.1 段位体系（自创 13 级，不直接用中羽 1-9）

| 段位 | 描述 | 中羽对照 |
|---|---|---|
| **D 级（青铜）**：D1/D2/D3 | 入门 | 1-3 |
| **C 级（白银）**：C1/C2/C3 | 业余中等 | 4-5 |
| **B 级（黄金）**：B1/B2/B3 | 业余高手 | 6-7 |
| **A 级（铂金）**：A1/A2/A3 | 业余顶尖 | 8 |
| **S 级（钻石）**：S | 接近退役专业 | 9 |

### 8.2 评测题目（共 20 题，5-7 分钟）

| 类型 | 题数 | 权重 |
|---|---|---|
| 球龄/打球频率 | 2 | 客观信息 |
| 技术动作（多选/单选） | 8 | 主要权重 |
| 战术意识（情境题） | 4 | 高段位关键 |
| 比赛经验 | 3 | 段位定锚 |
| 自评 + 动作视频自检 | 3 | 校准用 |

评分算法见 [6-核心算法.md §4](6-核心算法.md)。

### 8.3 报告卡片（裂变核心）

**第一页**：段位徽章 + 标题（"小区野球王"）+ 综合分数 + 一句话评价

**解锁详细分析**：分享朋友圈解锁，看到 9 张精美图：
1. 技术雷达图（6 维：攻/防/网前/底线/步伐/控球）
2. 同段位排名
3. 进阶建议
4. 推荐课程（**跳转到"教学"Tab 对应内容**）
5. 历史成长曲线

**分享文案**：「我刚测出来是 **B2·黄金双星**「小区野球王」🏸 你也来测测 ↓↓」

### 8.4 防作弊
- 同一微信号 30 天只能测 1 次
- 重测要扣"挑战币"（每周送 1 个）

### 8.5 AI 教练问答彩蛋（v1 新增）

> 段位报告页底部"🤖 让 AI 教练看一眼"按钮。**不参与段位评分**。

**3 种交互**：
- 📸 看我的握拍（拍图 + 可选文字）
- 📸 看我的站位
- 💬 自由问问题

## 功能四：训练成长

> **留存第一支柱：把"测完段位就走"变"每天回来打卡"。**

### 9.1 设计目标
段位评测 → 个性化训练计划 → 打卡跟练 → 复测看进步，形成完整闭环。
**训练菜单中的视频内容全部来自"教学"Tab 内容库**，复用同一套内容。

### 9.2 技能树（4 大类约 30 个技能）

```
基础类：握拍 / 发球（高远/网前/平射/反手）/ 步伐（米字步/交叉步/并步/跨步）
进攻类：杀球（直/斜/点杀/重杀）/ 劈吊（正/反/滑板）/ 平高 / 平抽挡
防守类：接杀（挡网/抽直线/勾对角）/ 放网 / 挑高球 / 接吊
战术类：双打站位（前后/平行/轮转）/ 单打跑动 / 心理对抗
```

每个技能挂载：段位要求、训练动作（如杀球连续 50 个×3 组）、教学视频外链、自查清单、难度。

### 9.3 训练计划生成（算法见 [6-核心算法.md §10](6-核心算法.md)）

**输入**：当前段位、目标段位、雷达图弱项、训练频率（默认每周 2 次×1h）、场地（家/球馆有无陪练）、周期（4/8/12 周）

**输出（4 周示例）**：
- 第 1 周：基础打牢（反手挑球、米字步）
- 第 2 周：核心进攻（杀球落点、劈吊）
- 第 3 周：组合应用（高远转杀球、杀球+网前推扑）
- 第 4 周：实战+复测

### 9.4 打卡机制

| 触发 | 操作 | 反馈 |
|---|---|---|
| 完成单项训练 | 点 ✅ | +5 积分 + 进度条 |
| 看完视频 | 自动检测/手动点 | +2 积分 |
| 连续打卡 | 系统记录 | 7 天 +20 / 14 天 +50 / 30 天 +200 |
| 漏卡 | 系统检测 | 微信订阅消息提醒 |

### 9.5 周报与复测
- 每周日推送周报
- 4 周计划结束自动弹"复测段位"
- 复测对比：4 周前 C2·65 分 → 现在 C3·78 分
- 强烈鼓励分享对比海报到朋友圈

### 9.6 商业化路径（v1 全免费）

| 阶段 | 模式 | 价格 |
|---|---|---|
| v1 | 全免费 | ¥0 |
| v1.5 | 进阶会员 | ¥19-29/月 |
| v2 | 单课购买 | ¥9.9-99/课 |
| v3 | 1v1 教练咨询 | 抽佣 15% |

### 10.8 H5 推文引流策略（v1 重点）

H5 站点不只是内容承接，更是**站外流量入口**。每月产出 5-10 篇精选推文，分两类：

#### 类型 A：长尾 SEO 推文（百度/Bing 引流）
覆盖业余球友常搜的问题，标题直接含关键词：
- 「羽毛球拉多少磅合适？2026 业余球友最全指南」
- 「网前球总下网？5 个动作问题自查清单」
- 「30 元一筒的球和 80 元的差在哪？」
- 「业余球友买第一支拍，1000 元价位 3 选 1」

#### 类型 B：种草破圈推文（小红书/朋友圈传播）
做情感共鸣，不堆参数：
- 「我打球 10 年，终于明白业余球员最该练什么」
- 「我们小区球队的故事：从 5 人到 20 人」
- 「测了段位之后，我才知道自己原来这么菜」

#### 引流闭环
```
百度/小红书/抖音搜索
  → 落到 H5 文章
  → 文章末尾"用 YMQ 测段位 / 找球友 / 排对阵"
  → 引导扫码或跳小程序
  → 在小程序里完成核心动作（测段位/创建活动）
```

#### 内容质量门槛
- 精选 > 数量：宁可一个月 5 篇精品，不要 50 篇水文
- 每篇必须有"能解决一个具体问题"的承诺
- 装备类推文 v2 起埋 CPS 联盟链接（京东/淘宝联盟）做变现

### 10.9 赛事/动态分类（v1.3 新增，不做独立资讯 Tab）

> **明确决策**：不做独立"资讯 Tab"。羽毛球资讯有更专业的产品（虎扑、新浪体育），用户不会来 YMQ 看新闻。

**取而代之**：在教学 Tab 下加"赛事/动态"分类，每周更新 2-3 条精选。

#### 内容类型

| 类型 | 频率 | 来源 |
|---|---|---|
| 大赛事点评 | 大赛期间每天 1 条 | 手写：汤尤杯/全英赛/世锦赛点评 |
| 装备新品速递 | 每月 1-2 条 | YONEX/Victor/李宁新品发布 → 含 CPS 链接 |
| 本地培训/比赛信息 | 每月 2-3 条 | 张家港 + 周边俱乐部、培训班合作 |
| 球友圈热点 | 每月 1-2 条 | 球友群讨论 / 装备种草 / 趣事 |

#### **绝对不做的事**

- ❌ AI 全自动爬虫抓取（成本高、质量差、版权风险、维护累）
- ❌ 转载央视/新华社/虎扑文章（版权问题）
- ❌ 实时新闻推送（不是新闻 App）
- ❌ 资讯独立 Tab（稀释主线 5 个功能）

#### 制作流程
- 每周一花 30-60 分钟手写或 AI 辅助生成 2-3 条
- 装备类要含具体使用体验（不只是参数堆砌）
- 本地类靠合作俱乐部主动供稿，不靠爬取
- 大赛事用 Claude 看赛事直播片段后辅助生成点评（人工最终把关）

## 11. 种子市场策略（v1.3 新增）

> 独立开发者不能"全国铺开"——必须有明确的种子市场聚焦。

### 11.1 首发城市：张家港 + 周边（苏州/无锡/常州）

#### 为什么是张家港
- 团队所在地，本地资源最熟
- 县级市规模适中：球友圈足够密集，竞争对手少
- 江苏整体经济好，业余羽毛球氛围浓
- 张家港本地无强势竞品

#### 为什么不是上海/北京/深圳
- 大城市球友圈高度分散，难形成密度
- 已有约球类 App 渗透深
- 推广成本几何级数高
- 球友对工具型小程序需求弱（他们用专业 App）

### 11.2 首批种子用户（30-50 人，1-2 个月内拿下）

| 来源 | 目标数 | 方法 |
|---|---|---|
| 团队自己的球队 | 1 个（5-10 人） | KOC #1，自用打磨 |
| 张家港主流羽毛球微信群 | 3-5 个（30-50 人） | 联系群主试用 + 反馈 |
| 张家港体育中心固定球友 | 1-2 个球队 | 线下扫码 + 试用礼包 |
| 本地俱乐部合作 | 1-2 家 | 提供活动管理工具换信息 |

### 11.3 本地化产品配置

#### 内置场馆词典（v1 MVP 必做）

张家港主要羽毛球场馆（运营后台预置）：

### 11.8 冷启动 3 步走战术（首发 0-3 月）

  - 哪些群活跃（每周活动数）
  - 谁是群主（潜在 KOC）
  - 组织活动的具体痛点（接龙怎么排）
  - 用户对现有工具（群接龙、约球 App）的不满

→ **混群本身比做产品更重要**：MVP 上线时你应该已经认识了 10 个潜在 KOC 群主。

#### 第 1 步：MVP 灰度给 10 个 KOC 群主试用（MVP 上线后 1-2 月）

- 一对一加微信，亲自上门或视频演示
- 每周一对一收反馈，迭代上线
- 给群主额外特权：
  - 看自己群的战绩榜
  - 看群成员段位分布
  - AA 收款二维码
  - 群活动一键迁移到 YMQ
- 不要求他们传播，让产品体验自己说话

→ KOC 用一周后会主动想分享给球友——这才是真正的传播起点。

#### 第 2 步：内容矩阵 + 公开推广（2-3 月开始）

| 渠道 | 内容方向 | 频率 |
|---|---|---|
| **抖音** | 30 秒短视频："这个小程序排对阵真省事"、"今晚一键凑齐 6 人局"、"AI 看我握拍说我错了" | 每周 2-3 条 |

| 教学内容收藏达 10 篇 | "我的羽毛球学习清单" |
| H5 优质推文 | 站外搜索/小红书传播 → 引流回小程序（详见 §10.8） |

### 12.3 海报模板系统
30+ 套模板，覆盖：接龙卡片（5 套）/ 对阵表（5 套）/ 战报（10 套）/ 段位报告（5 套）/ 颁奖（5 套）。

### 12.4 排行榜玩法（v2）
小区周排行 / 球队周排行 / 全国段位排行。

### 12.5 邀请激励（v2）
邀请 3 朋友测段位解锁 1 张"挑战券" / 邀请新人成立球队解锁球队主页定制。

## 非功能性需求

### 13.1 性能
- 小程序首屏 ≤ 1.5s
- 接龙报名 ≤ 300ms
- 对阵表生成 ≤ 2s（10 人以下）
- 实时比分同步 ≤ 1s
- 发现页查询 ≤ 500ms
- 教学 Tab 列表加载 ≤ 800ms

### 13.2 安全
- 微信登录 + JWT
- 所有 API 鉴权（除公开活动详情和公开内容详情）
- 输入校验防 SQL 注入、XSS
- 敏感操作（删活动、移除人）需二次确认
- 防接口刷：限流 + 滑动验证
- 位置数据加密存储

### 13.3 合规
- 用户协议 + 隐私协议（小程序审核要求）
- 信息最小化：openid + 昵称 + 头像 + 性别（选填）
- 位置信息单独授权
- **小程序类目：体育→运动健身**（不申请"社交"）
- 不存储微信支付敏感信息
- ICP 备案

### 13.4 监控
- 业务监控：活动创建数、用户增长、分享次数、教学内容阅读数
- 技术监控：API 响应时间、错误率、服务器资源
- 错误追踪：Sentry

## AI 能力路线

### 14.1 原则
- **分阶段**：不一次性 all-in AI
- **不影响段位**：AI 永远不替段位评分（避免投诉）
- **裂变优先**：每个 AI 功能都要有分享钩子
- **成本可控**：单次调用透明，配额防滥用
- **可回退**：AI 服务故障不影响核心功能

### 14.2 四阶段

| 阶段 | 时间 | AI 能力 | 用户规模门槛 | 开发成本 | 月运营成本 |
|---|---|---|---|---|---|
| **v1** | Sprint 7-8 | AI 教练问答彩蛋（图+文） | 0 | 3-5 天 | ¥500-2000 |
| **v1.5** | MAU 3000 后 | 多轮对话 + 个人档案 + 推荐课程 | 3000 | 2-3 周 | ¥2000-8000 |
| **v2** | MAU 5w 后 | 单动作姿态分析（MediaPipe BlazePose + 规则引擎） | 50000 | 6-8 周 | ¥5000-2w |
| **v3** | MAU 20w 后 | 长视频段位评估（综合评分给"参考段位"） | 200000 | 6-12 个月 | ¥3w-10w |

### 14.3 v1 AI 教练彩蛋的技术细节

**API 选型**：Claude 4.x（首发） + 国内热备（豆包/Qwen-VL，环境变量切换）

**Prompt 模板**：
```
你是"小羽教练"，一位耐心的业余羽毛球教练。
用户上传了一张关于「{type}」的图片。

请按以下格式输出：
1. ✅ 做得好的（1-2 点，鼓励为主）
2. ⚠️ 需改进的（1-3 点，具体可执行）
3. 💡 改进建议（1-2 句话）
4. 📺 推荐练习方向（不超 20 字）

要求：
- 称呼"球友"
- 语气友好不批判
- 图片看不清礼貌请重拍
- 不评段位、不和其他选手对比
- 不超过 200 字
```

**数据飞轮**：v1 起就存图片+回答+用户反馈，为 v2 训练做种子。

### 14.4 关键风险与应对

| 风险 | 应对 |
|---|---|
| AI 回答不准 | Prompt 严格约束，安全话术，用户反馈循环 |
| 用户用 AI 测段位（绕过答题） | 严格分离：AI 教练页永不出现段位文字 |
| API 成本失控 | 配额机制 + 监控告警，超预算降级到固定话术 |
| 敏感图片 | 微信内容安全 API 前置过滤 |
| 国际 API 中断 | 国内备用 API 一键切换 |

## 商业化路径

> v1 全部免费，先验证留存。本节梳理**所有可能的变现路径**，按时间先后排列，帮你看清"这个产品到底怎么挣钱"。

### 15.1 商业化原则

- **工具永远免费**：核心组织功能（创建活动/对阵表/接龙）不收费
- **内容/数据/抽佣三条腿**：变现走的是产品的副产品，不是工具本身
- **不卖用户隐私**：永不出售位置/通讯录/段位等数据
- **不接硬广**：教学/装备内容可植入但要明确标识

### 15.2 变现路径全景

| # | 变现方式 | 启动阶段 | 单笔/单次 | 月收入预估（MAU 1w 时）| 月收入预估（MAU 10w 时）|
|---|---|---|---|---:|---:|
| 1 | **装备 CPS 联盟**（京东/淘宝/抖音联盟） | v2（MAU > 3000） | 2-8% 分佣 | ¥1k-5k | ¥1w-5w |
| 2 | **付费会员**（解锁高级训练 + 独家课程） | v1.5（MAU > 3000） | ¥19-29/月 | ¥3k-1w（按 2% 付费率） | ¥3w-10w |
| 3 | **教练入驻分佣**（教练发课，平台抽 15%） | v2.5（MAU > 5w） | 课程价 ¥99-999 | ¥0-5k | ¥1w-10w |
| 4 | **场馆订场佣金**（接入场馆订场系统抽 5-10%） | v3（MAU > 5w） | ¥30-100/场 | ¥0 | ¥1w-5w |
| 5 | **赛事报名分账**（地方公开赛代收报名费） | v2.5（MAU > 5w） | 5-10% 抽成 | ¥0 | ¥5k-3w |
| 6 | **1v1 在线教练咨询**（视频咨询抽佣 15%） | v3 | 课时费 ¥100-300 | ¥0 | ¥5k-2w |
| 7 | **品牌联名/赞助**（YONEX/Victor/李宁等） | v3+（MAU > 10w） | 单次 ¥1w-50w | ¥0 | 不稳定 |
| 8 | **企业团建套餐**（公司组织羽毛球团建工具+场馆+教练） | v3+ | 单次 ¥2k-2w | ¥0 | ¥1w-5w |
| 9 | **数据报告**（给装备厂商出业余球友画像报告） | v3+（隐私合规前提） | 单份 ¥1w-10w | ¥0 | 偶发 |

### 15.3 变现节奏

```
v1（0-6 月）        ─ 完全免费，攒用户和留存数据
v1.5（6-12 月）     ─ 上付费会员（启动第一波变现）
v2（12-18 月）      ─ 上装备 CPS（启动第二波，被动收入）
v2.5（18-24 月）    ─ 上教练入驻 + 赛事报名（启动 B 端变现）
v3（2 年+）         ─ 上场馆订场 + 1v1 咨询 + 品牌赞助
```

### 15.4 各路径详解

#### 路径 1：装备 CPS 联盟（最早能见钱）
- **入口**：教学 Tab 的"装备指南"分类文章末尾，AI 教练回答末尾
- **形态**：联盟链接，用户点击跳京东/淘宝完成购买，平台拿 2-8% 分佣
- **关键**：内容质量要高，避免"低质种草号"
- **预期**：MAU 1w 时月入 ¥1k-5k；MAU 10w 时月入 ¥1w-5w

#### 路径 2：付费会员（最稳定）
- **入口**：训练成长进阶计划 + 独家课程
- **形态**：¥19/月 或 ¥199/年
- **付费率假设**：1-3%（业余羽毛球用户付费意愿低于健身用户）
- **关键**：免费版价值要足够，付费版要有明显增量（独家 + 数据深度）

#### 路径 3：教练入驻分佣
- **入口**：教学 Tab"教练入驻"专区
- **形态**：教练发布付费课程，用户购买，平台抽 15%
- **关键**：需要先有 5w+ MAU 才能吸引教练入驻
- **风险**：教练课程质量参差，要做审核

#### 路径 4：场馆订场佣金（最大想象空间）
- **入口**：活动创建时"订场馆"按钮
- **形态**：接入场馆订场系统（如约约球、好球管家），抽 5-10%
- **关键**：需要谈下足够多的场馆合作
- **风险**：场馆订场已有专业平台（如约约球），切入难

#### 路径 5：赛事报名分账
- **入口**：比赛 Tab"赛事日历"
- **形态**：地方公开赛、俱乐部赛事代收报名费
- **关键**：和本地俱乐部、体育局建立合作
- **机会**：张家港本地起步，能积累信用

#### 路径 6：1v1 在线教练咨询
- **入口**：AI 教练页面"找真人教练"按钮
- **形态**：用户付费预约教练视频咨询
- **抽佣**：15%
- **关键**：需要先有教练资源池（v3 阶段）

### 15.5 不做的变现方式

| 方式 | 不做的理由 |
|---|---|
| 工具收费 | 与"工具永远免费"原则冲突，会立刻流失用户 |
| 售卖用户数据 | 违法 + 信任崩塌 |
| 开屏广告 / 弹窗硬广 | 严重伤害体验 |
| 诱导分享换功能 | 微信小程序明确禁止 |
| 收购式购物（不是 CPS）| 涉及电商资质和退款合规 |
| 信息流广告 | 与"工具型产品"定位不符 |
| 用户提现/资金沉淀 | 涉及金融牌照 |

### 15.6 月收入推演（保守估算）

| 时间点 | MAU | 主要收入来源 | 月收入预估 |
|---|---|---|---|
| 6 月 | 1 万 | 完全免费 | ¥0 |
| 12 月 | 3 万 | 会员 + 少量 CPS | ¥5k-1.5w |
| 18 月 | 8 万 | 会员 + CPS + 少量教练分佣 | ¥3w-8w |
| 24 月 | 15 万 | 全路径启动 | ¥10w-30w |
| 36 月 | 30 万+ | 含场馆 + 赞助 | ¥30w-100w |

> 数字保守估算，实际取决于运营和市场。但**前 12 月不指望收入，专注用户和留存**。

## 产品风险

> 待决策问题已搬到 [3-路线图与待决策.md](3-路线图与待决策.md)。本节只列**产品级风险**。

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| 微信群已用群接龙，召集人不愿换 | 高 | 高 | 强化"接龙后直接出对阵表"的差异化 |
| 对阵表算法效果不佳 | 中 | 高 | 算法独立模块，支持手动调整 |
| 比赛记分使用频率低 | 中 | 低 | MVP 即可，不投入太多 |
| 段位评测被认为不准 | 中 | 中 | 强调娱乐性，引入互测 |
| 教学内容质量低 | 高 | 中 | 内容先少而精，月发 5-10 篇 |
| 发现附近用户密度不够 | 高 | 高 | MVP 不强推"发现附近"子页 |
| 召集人不愿开放公开活动 | 中 | 高 | 强化审核/信用/保护机制 |
| 女性用户安全感不足，流失 | 中 | 高 | 女性保护机制、举报响应 24h、首批种子女性占比 ≥ 30% |
| 训练计划完成率低 | 高 | 中 | 简化打卡 + 微信订阅消息提醒 |
| 训练计划见效慢（4 周练不出明显进步） | 高 | 中 | 用"打卡次数"代替"段位提升"作核心指标 |
| 被认定为"社交交友"类目 | 中 | 高 | 谨慎设计文案，禁止用户私聊，所有互动以活动为载体 |
| 没人愿意付费 | 中 | 低 | 工具免费，靠流量变现 |
| 被微信封禁 | 低 | 极高 | 严格遵守生态规则，不诱导分享、不刷量 |
