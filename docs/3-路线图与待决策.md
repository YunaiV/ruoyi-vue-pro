## 路线图原则

完整 PRD 中的六个月路线图保留为长期愿景，本文件把它调整为**更适合执行的阶段门禁**：

- 每一阶段都要验证一个核心假设
- 未过门禁，不自动进入下一阶段
- 新模块优先以**灰度实验**进入，不直接变成主导航和重投入

---

## 阶段 0：工程地基（第 1-2 周）

**目标**：跑通芋道骨架、小程序登录、基础模块结构和部署链路。

**交付**：
- Fork 并跑通 YunaiV/ruoyi-vue-pro
- 移除 mall、erp、crm、bpm、ai 等无关模块
- 建立 `ymq-module-activity`、`ymq-module-poster`，其余业务模块可先只占位
- 跑通小程序登录和后端鉴权
- 建立 C 端用户表，与运营后台 `system_users` 隔离
- 建立最小 CI/CD 或可重复部署脚本

**门禁**：
- 本地和测试环境均可启动
- 小程序能登录并请求 `/app-api/ymq/**`
- 运营后台能访问 `/admin-api/**`

→ **逐项 checklist 见 [Sprint0实施清单.md](Sprint0实施清单.md)**

## 阶段 1：组织活动闭环 MVP（第 3-8 周）

**目标**：验证"微信群活动组织工具"是否成立。

**交付**：
- 活动创建、编辑、取消
- 活动报名、候补、手动占位、移除
- 活动报名分享卡片
- 对阵表生成算法
- 召集人手动调整
- 普通球友"我的对阵"
- 对阵表分享卡片
- 基础战报
- 运营后台活动和用户列表

**门禁**：

| 指标 | 通过标准 |
|---|---:|
| 种子球队 | 10 个 |
| 真实活动 | 20 场 |
| 创建后生成对阵表比例 | 80% |
| 召集人二次创建比例 | 50% |
| P0 故障 | 0 个未解决 |

→ **完整验收标准见 [2-MVP与验收.md](2-MVP与验收.md)**

## 阶段 2：发现附近活动实验（第 9-12 周）

**目标**：验证"缺人活动公开化"是否能提高满员率，而不是做泛社交。

**交付**：
- 活动公开开关
- 可见半径、段位限制、性别限制、陌生人上限
- 简化版发现列表
- 申请加入和召集人审核
- 最小信用分：默认 100、爽约扣分、准时加分
- 举报和黑名单最小版

**门禁**：

| 指标 | 通过标准 |
|---|---:|
| 公开活动占比 | 20% |
| 公开后成功补齐活动比例 | 30% |
| 附近申请通过率 | 50% |
| 女性用户投诉 | 0 个严重问题 |

**处理原则**：
- 如果公开活动列表经常为空，不把"发现"做成强 Tab
- 如果安全和骚扰问题早期明显，暂停扩展，先补信任机制

## 阶段 3：增长实验（第 13-18 周）

**目标**：验证段位评测和比赛记分哪个更能带来传播或高频使用。

> ⚠️ **不建议两条同时重投入**——独立开发者精力会被摊薄。

### 候选 A：段位评测
- 题库管理
- 20 题评测
- 13 级段位报告
- 分享卡片
- 技术雷达图
- AI 教练问答彩蛋可灰度

### 候选 B：比赛记分
- 接力赛 100 分制
- 淘汰赛 4/8/16 人
- 裁判端记分
- 观众端实时比分
- 颁奖海报

### 选择建议
- 如果现有用户主要是球队组织者 → 优先**比赛记分**
- 如果需要朋友圈破圈拉新 → 优先**段位评测**

**门禁**：

| 指标 | 通过标准 |
|---|---:|
| 分享打开率 | 25% |
| 新用户注册转化率 | 15% |
| 单次功能完成率 | 70% |

## 阶段 4：留存模块（第 19-24 周）

**目标**：验证用户是否愿意围绕训练计划持续回来。

**交付**：
- 技能树数据建模
- 4 周训练计划生成
- 每日/每周任务
- 训练打卡和积分
- 周报推送
- 复测对比海报
- 知识库 H5 作为内容承接
- 运营看板

**门禁**：

| 指标 | 通过标准 |
|---|---:|
| 训练计划生成到打卡转化率 | 40% |
| 4 周完成率 | 20% |
| 复测完成率 | 30% |
| 训练内容有用反馈 | 60% |

## 延后处理

以下能力保留在完整愿景，**不进入 6 个月内硬承诺**：

- 付费会员
- 教练入驻
- UGC 知识库
- 双败淘汰和循环赛
- ELO 深度战绩体系
- MediaPipe 姿态分析
- 长视频段位评估
- 独立 admin-server 和 app-server 拆分（MAU > 5w 再拆）

## 已明确的产品原则

| 决策 | 说明 |
|---|---|
| 不做泛陌生人社交 | 所有陌生人互动必须以活动为载体 |
| MVP 不做 AA 收款 | 避免支付、退款、纠纷和审核复杂度拖慢核心链路 |
| 算法必须可手动兜底 | 对阵表不是黑盒，召集人要能调整 |
| 发现模块不强推 | 只有当活动密度足够时，才把发现作为主入口 |
| AI 不参与段位评分 | AI 只做教练建议和彩蛋，避免评测可信度争议 |
| 小程序类目走"体育" | 不申请"社交·交友"类目（避免严审） |
| 训练计划 v1 全免费 | 先验证留存再谈付费 |
| 内容生产 0 成本起步 | MVP 训练内容 100% 整合公开教程外链 |
| **装备不独立成 Tab** | **保留在教学 Tab 的"装备指南"分类下，v1 写 5-10 篇精选；v2 接 CPS 联盟链接做变现；阅读占比 > 20% 再考虑独立化** |
| **H5 推文是引流主力** | **教学知识库的 H5 站点不只是内容承接，更是 SEO + 朋友圈/小红书破圈的主战场。每月产出 5-10 篇精选推文，覆盖长尾搜索词 + 情感共鸣两类** |
| **资讯不独立成 Tab** | **不做独立资讯 Tab，不上 AI 全自动爬虫（成本高/质量差/版权风险）。教学 Tab 加"赛事/动态"分类，每周手编/AI 辅助 2-3 条精选** |
| **种子市场聚焦张家港** | **不全国铺开。先在张家港攒 10 个种子球队，再扩苏州/无锡/常州，再扩江苏全省，最后全国。详见 [1-产品愿景.md §11 种子市场策略](1-产品愿景.md#种子市场策略)** |
